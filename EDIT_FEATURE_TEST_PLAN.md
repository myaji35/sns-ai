## 테스트 케이스

### TC-001: 수정 버튼 클릭 및 모달 열림
**목적:** 수정 버튼 클릭 시 모달이 정상적으로 열리는지 확인

**테스트 단계:**
1. http://localhost:3010/management 페이지 접속
2. 회원사 목록에서 임의의 회원사 선택 (예: "맛있는 베이커리")
3. 해당 회원사의 "수정" 버튼 클릭

**예상 결과:**
- ✓ "회원사 수정" 제목의 모달이 화면 중앙에 표시됨
- ✓ 모달 배경이 반투명 검은색 오버레이로 덮임
- ✓ 모달 내부에 폼 필드들이 표시됨

---

### TC-002: 폼 필드 기존 데이터 확인
**목적:** 선택한 회원사의 데이터가 폼 필드에 정확히 표시되는지 확인

**테스트 단계:**
1. "맛있는 베이커리" 회원사의 수정 버튼 클릭
2. 모달 내 모든 폼 필드 값 확인

**예상 결과:**
- ✓ 회원사명: "맛있는 베이커리"
- ✓ 이메일: "bakery@example.com"
- ✓ 사업자번호: "123-45-67890"
- ✓ 사업자 유형: "소상공인" (small_business)
- ✓ 산업/업종: "음식/음료"
- ✓ 구독 플랜: "Starter"
- ✓ 사용자 수: "3"

---

### TC-003: 단일 필드 수정 (회원사명)
**목적:** 회원사명만 변경하고 저장했을 때 정상 작동하는지 확인

**테스트 단계:**
1. "맛있는 베이커리" 수정 모달 열기
2. 회원사명을 "최고의 베이커리"로 변경
3. "저장" 버튼 클릭

**예상 결과:**
- ✓ "회원사 정보가 수정되었습니다." 알림 표시
- ✓ 모달이 자동으로 닫힘
- ✓ 회원사 목록에서 이름이 "최고의 베이커리"로 변경됨
- ✓ 다른 필드 값들은 변경되지 않음

---

### TC-004: 복수 필드 수정
**목적:** 여러 필드를 동시에 변경했을 때 모두 정상 반영되는지 확인

**테스트 단계:**
1. "패션 부티크" 수정 모달 열기
2. 다음 필드들을 변경:
   - 회원사명: "프리미엄 패션 부티크"
   - 사업자번호: "999-99-99999"
   - 사업자 유형: "소상공인" (individual → small_business)
   - 구독 플랜: "Business" (free → business)
   - 사용자 수: "5" (1 → 5)
3. "저장" 버튼 클릭

**예상 결과:**
- ✓ 모든 변경사항이 회원사 목록에 정확히 반영됨
- ✓ 통계 카드도 함께 업데이트됨:
  - 개인 사업자: 1 → 0
  - 소상공인: 2 → 3
  - 총 사용자 수 증가 (1→5, +4명)

---

### TC-005: 사업자 유형 변경 시 통계 업데이트
**목적:** 사업자 유형을 변경했을 때 대시보드 통계가 정확히 업데이트되는지 확인

**테스트 단계:**
1. "테크 스타트업" 수정 모달 열기 (현재: 중소기업)
2. 사업자 유형을 "소상공인"으로 변경
3. "저장" 버튼 클릭
4. 상단 통계 카드 확인

**예상 결과:**
- ✓ 소상공인: 2 → 3
- ✓ 중소기업: 1 → 0
- ✓ 총 회원사: 4 (변동 없음)
- ✓ 총 사용자: 21 (변동 없음)
